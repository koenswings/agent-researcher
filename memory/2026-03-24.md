## Key lessons learned today

1. **Don't assume permissions without testing.** I told Koen I couldn't edit my own AGENTS.md because it was root-owned тАФ but I'd been editing root-owned files in other workspaces all day. Wasted time and credibility.

2. **Read the design doc at session start.** Made multiple mistakes (wrong paths, wrong ports, missing context) because I didn't read the virtual-company-design.md. Now fixed in AGENTS.md.

3. **ESM modules don't use NODE_PATH.** Tried NODE_PATH workaround for the md-to-pdf skill тАФ doesn't work for ESM. Symlinks work; `.mts` extension forces ESM mode in tsx.

4. **git commits need user.email/name per repo.** When committing in repos owned by different users, always set `git config user.email` + `git config user.name` locally first, or get "Author identity unknown".

5. **Don't include .env in git commits.** Nearly leaked GITHUB_TOKEN тАФ GitHub push protection rejected it. Always check `git status` before committing in agent workspaces.

6. **Telegram bot token for direct API calls:** extract from `/root/.openclaw/openclaw.json` тЖТ `channels.telegram.botToken`. The `message` tool `filePath` doesn't resolve container paths; use `curl -F document=@/path` directly.
