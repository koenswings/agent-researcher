### 1. Architecture Clarification

The pre-configured OpenClaw agents (`engine-dev`, `console-dev` etc.) and the MC board lead agents are **separate agent instances**:

### 2. MC Onboarding Flow

For each of the 5 boards:
1. `POST /api/v1/boards/{board_id}/onboarding/start` — created onboarding session
2. `POST /api/v1/boards/{board_id}/onboarding/confirm` — confirmed with objective + success metrics

This triggered `ensure_board_lead_agent` which:
- Created an MC agent record (`is_board_lead=True`)
- Registered the agent in OpenClaw (`openclaw.json`) as `lead-{board_id}`
- Provisioned workspace files: `AGENTS.md`, `BOOTSTRAP.md`, `IDENTITY.md`, `SOUL.md`, `TOOLS.md`, `USER.md`, `MEMORY.md`, `HEARTBEAT.md`
- Sent wakeup message: `"Hello {Name}. Your workspace has been updated. Start the agent. If BOOTSTRAP.md exists, read it first..."`
